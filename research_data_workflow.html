Case Study: Reproducible R-Based Data Analysis

Evidence type: Self-directed technical project using open data
Topic: Emotion regulation, stress, and symptom reporting

Goal

Build a complete, reproducible analysis rather than a one-off statistical output. The project needed a clean analysis dataset, documented decisions, interpretable models, diagnostics, and material that another person could follow.

Process

  • Started with 254 records.
  • Checked variables, missingness, coding, and analysis eligibility.
  • Produced an analysis sample of 247 records.
  • Tested descriptive and correlational relationships.
  • Built three staged regression models to compare demographic, psychological, and emotion-regulation predictors.
  • Documented limitations and non-significant findings instead of hiding them.
  • Prepared a README, analysis script, figures, tables, and short report.

Selected results

  • The analysis sample contained 247 participants.
  • Three regression models were compared.
  • Gender, anxiety, and depressive symptoms contributed to symptom reporting in the staged models.
  • The added emotion-regulation score was not significant in the final model.

Relevance to support

This project demonstrates the same habits required in technical support:

  • inspect the data before drawing a conclusion;
  • make assumptions explicit;
  • preserve reproducibility;
  • report negative or ambiguous results honestly;
  • separate the customer's symptom from the underlying cause;
  • document work so another person can continue it.
