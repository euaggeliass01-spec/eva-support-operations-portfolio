My troubleshooting method

A consistent process helps me stay useful when a customer is frustrated, information is incomplete, or a change could affect revenue or data.

Clarify impact

Who is affected, what changed, and how urgent is the business consequence?

Collect evidence

Exact error, timestamps, order IDs, logs, screenshots, environment, and recent changes.

Reduce scope

Reproduce safely, compare working and failing states, and change one variable at a time.

Communicate next step

Explain what I know, what I do not know, and what will happen next.

Document and prevent

Record the cause, resolution, limitations, and any product or process improvement.

AI-assisted support workflow

This prototype uses AI as a constrained analysis layer rather than an automatic answer machine. It supports prioritization and drafting while preserving human review for financial, privacy, and outage-related cases.

Working prototype

What the workflow does

  • Reads support tickets from CSV.
  • Redacts obvious email addresses and phone numbers before optional API use.
  • Returns structured category, priority, sentiment, risk flags, missing information, recommended actions, and a draft response.
  • Works in deterministic local mode without an API key.
  • Optionally uses the OpenAI Responses API with schema-constrained output.
  • Exports machine-readable JSON and a human-review HTML report.
Review sample output
Learning evidence

What I learned

  • A fluent draft can still contain an unsafe assumption.
  • Structured output improves consistency but does not remove the need for verification.
  • Financial and outage cases require explicit human-review rules.
  • Prompts work better when they define what the system must not claim or promise.
  • A local fallback makes the workflow auditable and usable when an AI service is unavailable.
